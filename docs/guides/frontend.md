# 前端开发指南

## 定位

前端不是单纯聊天页面，而是 Pony Agent 的“运行时调试台”。

## 推荐技术选择

- Vue 3
- TypeScript
- Pinia
- Vite

## 为什么现在建议上 Pinia

因为很快会出现这些全局状态：

- 当前会话
- 当前消息流
- 当前运行阶段
- 工具调用列表
- Provider 状态

如果没有集中状态管理，后续组件一多会很乱。

## 为什么现在不急着上 vue-router

第一阶段大概率只需要一个主工作台：

- 左侧聊天
- 中间主输出
- 右侧运行轨迹 / 工具日志

这更像一个工作台，而不是一个网站。

## 页面建议

### Chat Panel

- 用户输入
- 消息列表

### Runtime Panel

- 当前阶段
- 当前 provider
- 当前 turn 状态

### Tool Panel

- 工具名
- 参数
- 结果
- 时长

### Graph Trace Panel

- 当前节点
- 历史轨迹

## 前端最佳实践

- 用组件边界映射领域边界
- 用 store 管理领域状态
- 用 UI 可视化帮助理解智能体行为
