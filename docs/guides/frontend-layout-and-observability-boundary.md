# 前端分栏与可观测性边界

## 目的

这份文档用于固定 Pony Agent 首页工作台的信息架构，避免后续为了安放 `retrieval`、`context`、`memory`、`trace` 等信息，误改已经稳定的导航和对话结构。

这是增量开发约束文档，不是重做工作台的邀请。

## 首页三栏职责

- 左侧边栏：菜单导航、会话入口、会话历史、模型管理。
- 中间区域：对话主工作区，只承载消息流、输入区和与当前对话直接相关的主舞台内容。
- 右侧边栏：可观测性区域，承载 `Status`、`Tools`、`Trace`、`Retrieval`、`Diagnostics` 等运行态信息。

## 左侧边栏冻结结构

`HomeSessionSidebar` 一旦处于可用状态，默认视为冻结表面。未被任务明确邀请，不允许调整菜单顺序、命名、分组或交互语义。

当前固定结构如下：

1. 顶部品牌入口：`Pony Agent`
2. 主操作：`新对话`
3. 折叠菜单：`对话历史`
4. 折叠菜单：`模型管理`
5. `模型管理` 子项：
   - `模型配置`
   - `模型监控`

补充约定：

- 左侧边栏不显示单独的“主页”按钮。
- 点击 `Pony Agent` 返回对话工作台主页。
- 点击 `对话历史` 时，如果当前不在对话主页，应先返回对话主页，再展开或显示历史列表。

## Retrieval 的归属

`retrieval` 属于可观测性事实，不属于左栏导航，也不属于中间对话主舞台。

它回答的是：

- 系统本轮实际取用了哪些上下文。
- 当前 run 依赖了哪些 `history / attachment / memory / task facts`。
- 这些事实如何支撑当前的 `trace / handoff / continue`。

因此默认归属：

- 不放左侧导航区。
- 不常驻中间对话顶部。
- 放在右侧可观测性区域。

## Retrieval 的推荐表现

优先顺序如下：

1. 优先整合进右侧 `Trace` 面板中的紧凑分区。
2. 如果信息量和交互明显独立，再拆成右侧单独面板。

状态面板和 retrieval 不能混为一谈：

- `Status` 负责表达本次对话或本次运行的整体状态。
- `Retrieval` 负责表达本次运行到底取用了什么上下文事实。

## 禁止事项

未被任务明确邀请时，不允许因为新增 retrieval 或 observability 信息而做以下改动：

- 改写左侧边栏菜单结构。
- 改写中间对话工作台结构。
- 覆盖原有右侧 `Status / Tools / Trace` 主体实现。
- 顺手重排已稳定菜单顺序。
- 把“临时设计想法”直接落成全局结构调整。

## 允许事项

在增量开发中允许的动作：

- 在右侧 observability 结构内新增紧凑 retrieval 分区。
- 为冻结的信息架构补充契约测试。
- 在不改变既有结构语义的前提下修复交互错误。
- 在文档中补充新的边界和禁止事项。

## 开发执行规则

后续任何首页前端任务，在动手前都先检查：

1. 这次修改是否触碰左栏冻结结构。
2. 如果触碰，任务是否明确邀请了结构调整。
3. 新增内容是否本质上属于可观测性信息。
4. 如果属于可观测性信息，是否应优先进入右栏。
5. 是否已经补上对应契约测试，防止后续回归。

## 适用范围

本规则默认适用于：

- `src/components/HomeSessionSidebar.vue`
- `src/components/HomeWorkspace.vue`
- `src/components/HomeSidebar.vue`
- 首页后续新增的 `context / retrieval / diagnostics` UI
