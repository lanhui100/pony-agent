# 架构总览

## 目标

Pony Agent 的目标是构建一个可观察、可扩展的桌面智能体系统，而不是简单的聊天壳。

## 总体分层

### 1. Frontend

职责：

- 展示聊天与运行状态
- 展示工具调用和 Graph 轨迹
- 作为调试台承载交互

建议栈：

- Vue 3
- TypeScript
- Pinia

### 2. Tauri Shell

职责：

- 承载桌面窗口
- 提供前后端命令桥接
- 管理桌面应用生命周期

### 3. Rust Runtime

职责：

- 接收用户输入
- 组织单轮或多轮执行
- 调用模型
- 决定是否调用工具
- 管理执行状态

### 4. Provider Layer

职责：

- 封装不同模型接口
- 对外暴露统一调用协议

### 5. Tool Layer

职责：

- 管理工具定义
- 路由工具调用
- 返回结构化工具结果

### 6. Session / Memory Layer

职责：

- 管理会话上下文
- 做摘要
- 提供持久化记忆入口

## 与 Hermes 的关系

- `hermes/` 是学习对象，不是主开发区
- Pony Agent 的目标是提炼 Hermes 的有效经验，并用更清晰的 Rust 分层重建
- 对 Hermes 的借鉴以“思想迁移”为主，不以“代码搬运”为主
