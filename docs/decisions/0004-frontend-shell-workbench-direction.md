# 0001 前端工作台先采用原生 TypeScript + Vite 壳层

## 背景

当前工作区并不是完整的 Vue 版多面板工程，而是一版更轻量的 `TypeScript + Vite + Tauri` 壳层。用户当前的核心诉求也不是立刻补齐复杂前端框架能力，而是先把桌面端壳层、Rust 桥接和学习式界面跑通。

## 约束

- 需要优先保证 Tauri 窗口可启动、可展示、可验证
- 用户正在学习 Rust 和 agent 基础，需要界面足够直观
- 当前工程结构较轻，直接切大规模框架迁移会打断验证节奏

## 决策

当前阶段先保持原生 `TypeScript + Vite` 前端壳层，在此基础上继续迭代中文教学式工作台界面，并通过 Tauri `invoke` 打通 Rust 状态读取。

图标系统采用 `lucide`，布局以“对话区为主、辅助信息为辅”为核心原则。

## 权衡

### 好处

- 改动面更小，方便快速验证 Tauri 与 Rust 链路
- 学习成本更低，便于先理解页面结构与状态流
- 可以尽快把注意力放回 `run_turn()` 和运行时设计

### 代价

- 当前组件复用性和状态管理能力比较基础
- 如果后续需要复杂面板、路由和状态联动，仍可能升级到 Vue 组件化方案

## 后续影响

如果后续确认前端要承载更完整的 agent 工作台，再评估迁移到 `Vue 3 + Pinia`。在那之前，文档和实现都应承认当前状态是“轻量壳层”，避免描述超前于代码事实。
