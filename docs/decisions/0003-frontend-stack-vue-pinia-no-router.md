# ADR 0003：前端使用 Vue 3 + Pinia，暂不上 vue-router

## 背景

当前前端骨架是 `Vite + TypeScript + 原生 DOM`，不适合长期承载智能体工作台。

## 候选方案

### 方案 A

继续使用原生 DOM 和手写状态管理。

### 方案 B

升级为 `Vue 3 + TypeScript + Pinia`，第一阶段暂不接入 `vue-router`。

## 权衡

方案 A 的优点是简单，但当工作台出现多个面板与全局状态时，会迅速失控。

方案 B 的优点：

- 组件边界清晰
- 全局状态有合理归宿
- 适合承载聊天、运行时、工具日志和轨迹展示

暂不上 `vue-router` 的原因是：

- 第一阶段更像单页工作台，而不是多页面网站
- 当前重点是智能体主链，不是页面导航

## 决策

采用 `Vue 3 + TypeScript + Pinia`。

第一阶段暂不接入 `vue-router`。

## 影响

- 前端将切换到 Vue 应用结构
- 状态管理将以 Pinia 为中心
- 如果后续出现多页面场景，再追加 router 相关 ADR
