# ADR 0001：双轨重构，保留 Hermes 作为参考实现

## 背景

Hermes 是一个大型 Python 智能体工程，具备复杂的主循环、工具系统、记忆与平台接入能力。

如果直接在 Hermes 上原地重写 Rust，会带来：

- 学习与迁移边界不清
- 历史复杂度过高
- 回归风险过大

## 候选方案

### 方案 A

直接在 Hermes 仓库主结构上做 Rust 替换。

### 方案 B

保留 Hermes 作为参考实现，在新主工程里逐步重构。

## 权衡

方案 A 的优点是代码看起来更集中，但缺点是：

- 旧系统和新系统会互相污染
- 不利于理解重构边界
- 很容易陷入兼容性泥潭

方案 B 的优点是：

- 旧世界和新世界分离
- 有利于学习式重构
- 迁移路径清晰

## 决策

采用方案 B。

将 Hermes 放在 `hermes/` 目录作为参考实现，Pony Agent 根目录作为新主工程。

## 影响

- 新功能默认不写入 `hermes/`
- 学习和参考时主要从 Hermes 提炼思想，而非直接搬代码
