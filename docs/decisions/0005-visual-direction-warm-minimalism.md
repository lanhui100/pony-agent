# 0002 第一阶段视觉方向采用方案 A

## 背景

前端工作台已经开始搭建，因此需要在项目初期先定义好稳定的视觉方向，避免页面越做越散。

用户已经明确偏好：

- 极微小圆角
- 极简风格
- 对话区为主体
- 中文教学式表达

随后又进一步确认配色方向为：黑白灰为主，仅使用极少的暖橙色作为强调色。

## 决策

Pony Agent 第一阶段采用“方案 A：研究工作台”。

具体落地为：

- 主气质：理性、克制、专业
- 主色基底：黑白灰中性色
- 强调色：少量暖橙色
- 圆角：极微小圆角，控制在 `4px - 8px`
- 层次建立方式：以背景明度、字重、字号和留白为主，不依赖明显边框和重阴影

## 为什么这么做

这种风格适合当前项目的双重目标：

- 它足够理性，适合 agent、runtime、trace、tool call 这类工程信息
- 它又保留少量暖橙色，避免界面过冷，能给学习过程一点引导感

如果一开始就采用过强的品牌色或更消费产品化的视觉语言，后续在状态面板、执行链路和调试信息变多时，很容易显得嘈杂。

## 权衡

### 好处

- 更适合长期使用的工作台气质
- 更容易承载高信息密度界面
- 有利于后续扩展 agent 专属视图，而不容易花哨失控

### 代价

- 第一眼吸引力不如高饱和视觉方案
- 对排版质量要求更高，因为层次主要靠字体和间距撑起来

## 后续约束

后续新增页面和组件时，应尽量遵循以下限制：

- 不随意引入新的高饱和主色
- 不扩大圆角风格
- 不依赖大面积阴影做分层
- 重要交互优先通过排版、位置和少量暖橙强调来表达
