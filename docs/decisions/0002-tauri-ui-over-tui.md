# ADR 0002：优先使用 Tauri UI 作为运行时测试界面

## 背景

Pony Agent 当前阶段的重点是理解和重构 Rust 智能体核心，而不是复刻 Hermes 的终端交互体验。

## 候选方案

### 方案 A

优先做 TUI。

### 方案 B

优先做 Tauri UI 调试台。

## 权衡

TUI 的优点：

- 适合远程终端场景
- 可以快速验证命令行流程

TUI 的缺点：

- 会额外消耗大量精力在终端输入输出、滚动、兼容性和键盘交互上
- 不利于同时展示 Graph、工具轨迹和运行状态

Tauri UI 的优点：

- 便于可视化 runtime 状态、工具调用和执行轨迹
- 更接近最终桌面产品形态
- 更适合学习式重构和调试

## 决策

当前阶段优先做 Tauri UI 调试台。

TUI 若需要，放到后续作为开发者辅助入口考虑。

## 影响

- 前端工作台优先支持调试信息可视化
- 暂不投入主要精力在 TUI 主交互层
