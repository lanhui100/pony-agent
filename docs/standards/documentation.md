# 文档维护规范

## 文档分工

- `AGENT.md`：项目事实、原则、阶段、技术栈
- `docs/architecture/`：系统分层和设计
- `docs/decisions/`：重要技术决策与权衡
- `docs/guides/`：开发指南
- `docs/learning/`：学习问题、系统化梳理、内容素材沉淀
- `docs/standards/`：规范约束
- `docs/roadmap/`：阶段路线图
- `management/task-system/`：计划、进度、续作、审核、归档

## 编写原则

- 先写结论，再写原因
- 每份文档尽量只回答一类问题
- 避免把对话记录原样塞进文档
- 文档要服务后续开发，而不是堆信息
- 决策文档必须写出背景、备选方案、权衡和结论
- 学习文档要优先记录“问题、结论、系统化梳理、可延展内容选题”

## 更新时机

以下变更后应更新文档：

- 技术栈调整
- 目录结构变化
- 架构边界变化
- 阶段目标变化
- 做出重大技术决策
- 任务系统结构或使用规则变化

## 索引要求

- 新文档默认要在 `docs/INDEX.md` 中登记
- 重要文档应互相链接
