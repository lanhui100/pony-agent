# 工程规范与最佳实践

## 总原则

- 小步前进
- 先清晰后复杂
- 先可运行后可扩展
- 先可观察后可优化

## 目录规范

- `hermes/` 只作参考
- 根目录工程是 Pony Agent 主实现
- 架构、规范和路线图写入 `docs/`

## 编码规范

- 优先可读性
- 避免过早抽象
- 一个模块只负责一件事
- 重要设计选择要写明原因

## 重构规范

- 不以“尽快复刻 Hermes 全量能力”为目标
- 先做最小主链
- 新增抽象时必须回答：
  - 它解决了什么问题
  - 它的边界是什么
  - 如果没有它会怎样

## 测试与验证

- 每完成一个阶段，至少保证一条主流程可运行
- UI 要能展示核心运行状态
- 不把“看不见的正确”当成完成
