# 0001 学习式重构的目标、已完成工作与下一步

## 问题

我们现在这个项目到底已经做了什么，下一步做什么，为什么这样做？

## 简短结论

当前不是一次性重写 Hermes，而是在保留 Hermes 参考源码的前提下，建立 Pony Agent 新主工程，并按“前端工作台 -> Rust 最小闭环 -> 工具与编排”的顺序逐步重构。

## 系统化梳理

### 1. 核心概念

学习式重构的重点不是快速复刻，而是：

- 理解旧系统在解决什么问题
- 把有效经验重新组织成更清晰的新结构
- 让每一步都可验证、可解释、可延续

### 2. 已完成什么

当前已完成：

- 建立 Pony Agent 根工程
- 将 Hermes 放入本地参考区并忽略版本管理
- 建立 Tauri + Rust 基础骨架
- 将前端升级为 Vue 3 + Pinia 工作台
- 建立 AGENT、文档索引、任务系统、ADR 和学习记录体系

### 3. 下一步做什么

下一步主线是 Rust 侧最小闭环：

- 定义 `run_turn()` 相关领域对象
- 打通前后端 turn 调用
- 在 UI 中展示真实运行时状态

### 4. 为什么这样做

因为如果一开始就试图重做：

- 多代理
- 记忆系统
- 全量工具链
- 全部 Hermes 功能

就会立刻失去边界，难以学习，也难以验证。

## 常见误区

- 误区 1：重构就是把旧代码搬成另一种语言
- 误区 2：先做一堆复杂能力，后面再补主链

## 后续值得继续学什么

- Rust 智能体运行时如何做状态建模
- 工具调用如何做统一抽象
- 多轮会话和记忆系统如何渐进引入
