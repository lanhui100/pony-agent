# 0001 空白页排查与当前 UI 方向

## 这次发生了什么

Tauri 窗口曾经出现“能打开但页面空白”的现象。后续确认前端页面已经能够正常显示，因此这次工作的重点转为两部分：

- 同步确认当前工作区的真实技术状态
- 在现有代码基础上把界面调整为更适合学习的形态

## 这次确认到的事实

- 当前工作区不是完整的 Vue 多面板工程
- 当前前端入口是 `src/main.ts`
- 当前样式入口是 `src/styles.css`
- 当前前端是原生 `TypeScript + Vite`
- 当前页面已经通过 `@tauri-apps/api/core` 调用 Rust 侧 `health_check`

这件事很重要，因为它提醒我们：做学习式重构时，必须先以代码事实为准，不能沿用上一轮上下文里的“想象工程状态”。

## 这次 UI 为什么这样改

用户提出了几个明确的界面要求：

- 使用 `lucide` 图标
- 极简风格靠背景、字体、字重、字体颜色区分，而不靠边框和阴影
- 圆角进一步缩小
- 对话区域是主体，其他学习或展示面板是辅助
- 整体表达更偏中文、教学式、便于理解

因此页面的调整策略是：

1. 把左侧主区改成“Agent 对话实验台”
2. 把运行状态、架构焦点、学习提示压缩到右侧辅助栏
3. 降低边框和阴影存在感，改为更轻的背景层次
4. 用更明显的标题层级、说明性文案和图标辅助扫读

## 你可以从这个例子学到什么

### 1. UI 主次不是“说出来”，而是“排版出来”

当我们说“对话区是主体”，真正落地时要靠这些手段：

- 面积更大
- 信息密度更高
- 标题更突出
- 操作入口放在主区
- 辅助信息退到侧栏

### 2. 极简不等于没有设计

极简风格不是把所有东西都删掉，而是减少视觉噪音，让层次靠更克制的手段建立：

- 背景深浅
- 文本颜色差异
- 字号与字重
- 留白与排列

### 3. 学习阶段的 UI 要服务理解

在学习 Rust 和 agent 时，界面最重要的不是“酷”，而是让你一眼看懂：

- 这里是输入区
- 这里是 agent 回复区
- 这里是状态信息
- 这里是未来会接 `run_turn()` 的入口

## 这次改了什么

- `src/main.ts`：重写页面结构，引入 `lucide` 图标，强化主区与侧栏布局
- `src/styles.css`：重写视觉语言，收紧圆角，弱化边框阴影，强化中文教学式层级
- `package.json`：新增 `lucide` 依赖

## 当前结论

当前这版界面已经更符合“学习式 agent 工作台”的方向，但它还是一个静态前端壳层。下一步如果继续推进，最自然的路径是把“开始一轮演示”按钮接到真实的 `run_turn()` 入口，让界面不只是展示结构，而是展示一次真实回合是如何执行的。
