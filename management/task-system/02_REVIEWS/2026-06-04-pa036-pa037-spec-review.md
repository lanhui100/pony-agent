## 审核结论

两张卡的拆分方向成立，但都需要进一步收紧边界：

- `PA-036` 应聚焦 terminal envelope parity 与 persisted truth-source verification
- `PA-037` 应聚焦既有控制合同的前端消费闭环，而不是重做一层新的状态机或后端命令

## 采纳修改

1. 收紧 `PA-036` 范围
   - 不再把目标表述成“扩 failed-path provider/tool/hook evidence 读面”
   - 改为“验证 failed / cancelled terminal trace 不丢既有 evidence 字段”
2. 固定 `PA-036` 的合同边界
   - 明确本卡不新增 lifecycle event contract
   - terminal envelope `eventType` 只复用 `PA-031` 已定义 canonical terminal event
   - 不扩 hooks evidence model，不重写 monitor 聚合维度
3. 把 `PA-036` 的前端验收改硬
   - 当 persisted trace 缺失 terminal envelope 时，前端只能展示 non-canonical/raw trace 信息
   - 不得产出 completed / failed / cancelled canonical metrics
4. 收紧 `PA-037` 的 CTA 触发条件
   - 不再使用“submission plan 存在时”这种过宽口径
   - 改为 paused run、recovery checkpoint，或 submission plan 明确为 non-default resume / continue path
5. 绑定 `PA-037` 的真相源映射
   - `historical / historical_dirty` 绑定 `historyCursorMode`
   - `paused` 绑定既有 run / checkpoint / submission-plan 事实
   - `recovery-capable` 必须由 `submissionPlan / checkpoint` 派生
6. 固定 `PA-037` 的 command 边界
   - `replay` 仅作为前端文案或入口，不新增 replay backend command
7. 收口 `PA-037` 的验证面
   - `HomeSessionSidebar` 负责 history status / degrade feedback
   - `HomeWorkspace` 与 `runtime-store` 负责 stop/resume/replay CTA 展示与派发回归

## 结果

- 上述修改已回写到 `PA-036` / `PA-037` 的任务卡、OpenSpec spec 与 tasks
- `PA-036` 已补记当前实现进度，便于继续做 roundtrip / monitor / 前端 read-plane 验证
- `PA-037` 现已具备更稳定的 UI scope，可在不重做后端合同的前提下继续实现
