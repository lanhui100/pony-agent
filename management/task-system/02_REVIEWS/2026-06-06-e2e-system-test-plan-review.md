# Pony Agent 端到端系统测试方案审核记录

## 审核对象

- [端到端系统测试方案](</C:/Users/HUAWEI/Documents/pony-agent/docs/guides/e2e-system-test-plan.md>)

## 审核方式

- 使用 `opencode/minimax-m3-free` 进行独立只读审核
- 审核重点：
  - 跨层风险覆盖是否充分
  - 执行顺序是否可落地
  - 可观测性、数据隔离、防脆弱设计是否缺失
  - 发布准入是否可执行

## 模型审核结论

- 总体判断：`需修改`
- 核心评价：
  - 方案骨架完整，方向正确
  - 但偏“愿景文档”，还不够“可执行规约”

## 主要问题

1. 缺少 Tauri / IPC 跨层统一追踪骨架，证据标准偏软。
2. 文档中“建议”表述过多，关键工具栈与执行边界没有完全收敛。
3. 缺少 flaky 用例治理、重试预算、隔离机制与稳定度指标。
4. 缺少环境隔离、数据命名空间、setup/teardown 与并发执行契约。
5. 发布准入虽有原则，但缺少 sign-off、回滚和发布后冒烟闭环。

## 采纳结论

以上 5 条问题全部采纳，并已回写到方案文档：

1. 新增跨层可观测性契约，强制 `run_id / trace_id / event_seq` 贯通前端、Tauri、Rust。
2. 将工具栈、分阶段范围与 `v1 不做` 边界改为强制条款。
3. 新增 flaky 治理机制，包括单用例重试上限、`quarantine` 清单与 7 日滚动阈值。
4. 新增数据隔离与清理契约，明确 setup/teardown 和并发执行约束。
5. 新增发布签核与回滚章节，补齐产物、角色和观察窗口要求。

## 结果判断

- 修订后，方案已从“完整”提升为“可执行”
- 后续可继续进入：
  - 用例清单细化
  - `tests/e2e/` 目录与夹具脚手架建设
  - 系统测试执行脚本落地
